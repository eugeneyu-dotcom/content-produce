# 部落格自動化更新操作手冊 (編輯人員專用)

這份手冊將引導你如何透過簡單的三個步驟，自動完成資料蒐集、文章撰寫、圖片生成以及網站上線的完整流程。

---

## 🚀 第一次使用必看的「環境設定與必備檔案」

在你第一次開始寫文章之前，請確保你已經從主管或技術人員那裡取得以下權限與檔案：

1. **取得完整專案程式碼**：
   - 請確保你的電腦已安裝 **Git**。
   - 將本專案的 GitHub Repository Clone（下載）到你的電腦中。裡面包含了核心的生文腳本 (`ai_agent.js`) 以及四個網站的原始碼。
2. **安裝 Node.js**：
   - 請至 [Node.js 官網](https://nodejs.org/) 下載並安裝。這是為了讓你的電腦能執行自動化腳本。
3. **取得機密金鑰檔案 (`API_Key`)**：
   - **這是最重要的檔案！** 裡面包含了呼叫 AI 與 Google 搜尋所需的金鑰。
   - 因為安全考量，這個檔案不會放在 GitHub 上。請向主管索取名為 `API_Key` 的文字檔，並將它**放在專案的最外層目錄**（與 `ai_agent.js` 放在同一個資料夾下）。

確認以上三點準備就緒後，你就可以開始每天的自動化更新了！

---

## 📝 日常更新工作流 (SOP)

### 步驟 1：企劃與派寫任務 (操作 Google Sheet)

我們所有的文章企劃都是在雲端的 Google Sheet 上統一管理的。

1. 打開團隊共用的 **N8N_work - Workflow_Config** 試算表。
2. 找到你要指派 AI 撰寫的那一行關鍵字任務。
3. **填寫 `Human_Context` (人類觀點)**：
   - 這是整篇文章的「靈魂」。請用 1~3 句話簡單寫下你希望這篇文章切入的角度、你的個人經驗、或是你想吐槽的點。
   - *範例：「最近一直夢到蛇，上網查都說會發財，但我明明就嚇得半死。我想用心理學的角度來吐槽這種傳統迷信，告訴大家這其實是潛意識焦慮的展現。」*
   - AI 會完全根據你的這段話，寫出一篇帶有「真人溫度」的文章，這對 SEO 與讀者體驗極度重要！
4. **啟動任務**：
   - 將該列的 **`Status`** 欄位文字改為 **`Active`**。

---

### 步驟 2：一鍵自動生文與生圖 (操作終端機)

設定好表單後，接下來就交給 AI 機器人處理了。

1. 在你的電腦上打開「終端機 (Terminal / Command Prompt)」。
2. 使用 `cd` 指令切換到本專案的資料夾（例如：`cd Desktop/Old_Content_Farm`）。
3. 輸入以下指令並按下 Enter：
   ```bash
   node ai_agent.js
   ```
4. **放著讓它跑**：
   - 程式會自動連上雲端抓取你剛才設定為 `Active` 的任務。
   - 它會開始自動搜尋最新資料 ➔ 撰寫 1500 字長文 ➔ 自動生成頭尾兩張精美圖片 ➔ 自動排版存檔。
   - 過程中你不需要做任何事，只需等待終端機顯示 `[Success] Article saved to ...` 的成功訊息即可（通常大約需要 2~3 分鐘）。

---

### 步驟 3：審閱與上線發布 (Git 發布)

文章與圖片都自動生成並排版好後，最後一步就是把它推上正式網站。

1. **(可選) 本地預覽**：
   你可以打開對應站點的資料夾（例如 `Dream/src/content/blog/`），用文字編輯器稍微檢查一下剛產出的 Markdown 檔案內容是否符合預期。
2. **一鍵推播上線**：
   回到終端機，依序輸入以下三行指令，將新文章推送至雲端：
   ```bash
   git add .
   git commit -m "Add new article"
   git push
   ```
   *只要成功 Push，遠端的 Vercel / Netlify 伺服器就會立刻自動開始建置網頁，大約 1 分鐘後新文章就會在正式網站上線了！*

3. **任務結案 (狀態回寫)**：
   文章上線後，請回到 Google Sheet：
   - 將剛剛那列的 `Status` 從 `Active` 改為 **`USED`**（避免下次被重複執行）。
   - 在 `Post_Url` 欄位填上該篇文章正式上線的網址，以利日後內部連結引用與追蹤。

🎉 **大功告成！你已經順利產出並發布了一篇高品質的圖文部落格！**
